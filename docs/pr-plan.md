# VoiceCal Agent PR 拆分计划

本计划用于保证每个 PR 都保持小粒度、可 review、可持续交付。每个 PR 都应确保主分支保持可运行状态，并避免无关重构。

## PR 1：初始化 monorepo 项目结构

- 创建根目录基础文档。
- 创建 `docs/`、`backend/` 和 `frontend/` 目录。
- 说明项目定位、技术栈规划、功能规划和后续交付计划。
- 不初始化 Spring Boot 项目。
- 不初始化前端项目。

## PR 2：初始化后端 Spring Boot 骨架

- 在 `backend/` 目录初始化最小 Spring Boot 3 项目。
- 配置 Java 17 和 Maven。
- 添加健康检查接口。
- 添加本地开发基础 profile。
- 不实现日历业务逻辑。

## PR 3：建设后端 API 基础能力

- 添加统一接口响应结构。
- 添加全局异常处理。
- 添加请求参数校验能力。
- 为响应格式和异常处理补充基础测试。

## PR 4：建立日程领域模型

- 添加日程事件实体。
- 添加 Repository 层并支持 H2 本地存储。
- 添加日期时间字段的基础校验规则。
- 确保开始时间必须早于结束时间。

## PR 5：实现日程 CRUD API

- 添加日程创建、查询、修改和删除接口。
- 删除行为保持显式，方便后续接入确认机制。
- 添加聚焦的 Service 和 Controller 测试。

## PR 6：接入 LangChain4j Tool Calling 基础能力

- 添加 LangChain4j 依赖和配置。
- 基于已有日程服务定义 Tool Calling 工具。
- 添加最小可用的 AI 对话接口。
- 保持 prompt 和 tool 描述简洁、可测试。

## PR 7：初始化前端 Vite 骨架

- 在 `frontend/` 目录初始化 React、Vite、TypeScript 和 Tailwind CSS。
- 添加项目脚本和基础页面外壳。
- 不实现最终 Dashboard UI。

## PR 8：实现前端 Demo Dashboard UI

- 构建主 Dashboard 布局。
- 添加语音输入区、文本兜底输入区、AI 回复区、日历可视化区、今日和本周日程区、操作日志区。
- 补充空状态、加载状态、错误提示和成功反馈。

## PR 9：打通前后端基础交互

- 将前端输入流程接入后端 AI 对话和日历 API。
- 添加 API client 工具和 TypeScript 类型定义。
- 仅在后端能力尚未完成时保留必要 mock 数据。

## PR 10：实现日程查询和每日摘要能力

- 添加今日日程和本周日程接口。
- 通过 AI 助手提供每日摘要能力。
- 为日期范围和时区相关逻辑添加测试。

## PR 11：实现冲突检测和空闲时间查询

- 添加日程冲突检测能力。
- 添加空闲时间查询能力。
- 将两个能力同时暴露给 API 和 LangChain4j 工具。
- 为时间重叠和边界场景添加测试。

## PR 12：实现提醒字段和 ICS 导出

- 添加提醒相关元数据。
- 支持按日程或日期范围导出 ICS。
- 为生成的 ICS 内容添加测试。

## PR 13：实现危险操作确认机制

- 为删除和高影响修改操作添加确认状态。
- 在后端和前端接入确认流程。
- 避免 AI 直接执行误删或误改操作。

## PR 14：Demo 打磨与交付收口

- 优化种子数据和演示流程。
- 打磨前端视觉细节。
- 补充最终 README 启动说明。
- 验证后端和前端本地启动流程。
